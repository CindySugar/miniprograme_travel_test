# 旅行AA记账小程序整体情况说明

## 1. 项目目标
本项目要做的是一个旅行场景下的 AA 记账小程序，核心流程是：

1. 创建旅行
2. 邀请并管理成员
3. 逐笔记录花销
4. 支持部分成员参与分摊
5. 旅行结束后自动计算每个人应补/应收金额，并给出转账建议

目标是让旅行结束后的账单清晰、可追溯、可结算。

## 5. 当前重要缺口
### 1) 前后端尚未真正打通
现在小程序页面仍在用本地 `travel-store` 运行，而不是直接调用 Django API。

### 2) 后端数据库仍是开发态配置
当前 Django 默认数据库是 SQLite，便于本地开发和验证。
计划里要求的 MySQL 还需要后续切换并补环境配置。

- `wx.login`
