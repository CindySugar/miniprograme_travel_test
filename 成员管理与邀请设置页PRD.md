# 猫猫鱼记账 - 成员管理与邀请设置页 PRD

## 1. 背景
这是微信小程序“猫猫鱼记账”中，创建组队完成后的成员管理与邀请设置页面。

该页面用于维护当前组队的成员身份、邀请开关、家庭分组以及成员的记账权限状态，承接“创建组队页”之后的协作管理流程。

## 2. 页面目标
- 让管理员可以维护成员名单与成员状态。
- 让管理员可以控制是否允许新成员继续加入组队。
- 支持提前配置家庭分组，以便结算时按家庭合并转账。
- 支持查看成员当前的认领状态、微信绑定状态和记账权限状态。

## 3. 页面定位与入口
- 页面定位：组队创建完成后的后续管理页。
- 所属链路：`创建组队 -> 邀请/补充成员 -> 管理成员与家庭 -> 后续记账与结算`
- 入口方式：
  - 从组队详情或设置入口进入
  - 由创建者/管理员在组队建立后进入
- 页面不是独立产品页，而是现有组队产品链路中的成员管理页。

## 4. 页面结构
页面按截图固定拆分为以下 5 个区块：

### 4.1 顶部说明区
- 说明管理员可维护预置成员、清除错误认领、禁用记账权限、移出组队。
- 说明管理员也可以修改自己的昵称和头像。

### 4.2 邀请设置区
- 区块标题：`邀请设置`
- 右侧状态：`邀请开启中`
- 核心设置项：`允许新成员加入`

### 4.3 家庭管理区
- 区块标题：`家庭`
- 右侧说明：`最终按家庭转账`
- 包含成员选择、家主选择、组成家庭、已有家庭展示与移除家庭操作。

### 4.4 添加成员区
- 区块标题：`添加成员`
- 右侧状态文案：`未绑定微信，待认领`
- 包含昵称输入框与添加按钮。

### 4.5 成员列表区
- 区块标题：`成员列表`
- 右侧数量展示：如 `4名成员`
- 展示所有当前成员卡片及各自状态、标签与操作。

## 5. 分区说明
### 5.1 顶部说明区
- 使用说明性文案，明确这是管理员操作区。
- 文案重点：
  - 可维护预置成员
  - 可清除错误认领
  - 可禁用记账权限
  - 可移出组队
  - 可修改自己的昵称和头像

### 5.2 邀请设置区
- 展示当前邀请状态，默认文案为 `邀请开启中`
- 核心开关项：`允许新成员加入`
- 配套说明文案：
  - 关闭后，未加入成员不能再通过分享卡片加入
  - 已加入成员不受影响

### 5.3 家庭管理区
- 用于把多个成员归为同一个家庭。
- 页面需支持：
  - 选择同一个家庭的成员
  - 指定一名家主
  - 点击 `组成家庭`
  - 展示已创建的家庭卡片
  - 支持移除已有家庭
- 区块内需要提示家庭的用途：
  - `结算时按家庭合并转账`

### 5.4 添加成员区
- 输入框占位文案：`输入成员昵称`
- 操作按钮文案：`添加`
- 添加的是临时成员身份，若未绑定微信，应显示为待认领态。

### 5.5 成员列表区
- 展示每位成员的成员卡片。
- 每张卡片需要至少包含：
  - 头像或首字头像
  - 昵称展示位或输入位
  - 身份标签
  - 账单数
  - 记账权限状态
  - 微信绑定状态
  - 操作按钮

## 6. 角色与权限
### 6.1 角色定义
- 第一版中，`管理员` 与 `创建者` 视为同一角色。
- 页面中可以同时展示 `管理员` 与 `创建者` 标签，但权限不拆分。

### 6.2 管理员权限
- 保存成员昵称
- 禁用成员记账
- 清除成员认领
- 移出组队
- 配置家庭
- 指定家主
- 控制是否允许新成员加入
- 修改自己的昵称和头像

### 6.3 普通成员权限
- 第一版不在本页展开普通成员的管理操作权限。
- 页面以管理员维护为主。

## 7. 状态设计
### 7.1 邀请状态
- `邀请开启中`
- `邀请关闭`

### 7.2 成员身份状态
- `自己`
- `创建者`
- `管理员`
- `家主`

### 7.3 微信绑定与认领状态
- `已绑定微信`
- `未绑定微信`
- `待认领`

待认领相关文案需与截图语义保持一致，例如：`未绑定微信，待认领`

### 7.4 记账权限状态
- `可记账`
- `认领后可记账`
- `禁用记账`

### 7.5 家庭操作状态
- 少于两名成员时，`组成家庭` 按钮不可用。
- 未指定家主时，`组成家庭` 按钮不可用。
- 满足成员数量和家主条件后，`组成家庭` 按钮可用。

## 8. 关键交互
- 点击邀请开关，可切换是否允许新成员加入。
- 输入成员昵称后点击 `添加`，可新增一名成员。
- 空昵称时不可添加成员。
- 在家庭管理区选择多个成员后，可继续指定家主。
- 至少选择两名成员并指定家主后，才能点击 `组成家庭`。
- 点击 `移除` 可删除已有家庭分组。
- 在成员卡片中可执行：
  - 保存昵称
  - 禁用记账
  - 清除认领
  - 移出组队

## 9. 业务规则
### 9.1 邀请规则
- 第一版仅包含“允许新成员加入”的开关与说明文案。
- 第一版不展开分享卡片、邀请码、认领链路的详细设计。
- 关闭邀请后：
  - 未加入成员不能再通过分享卡片加入
  - 已加入成员不受影响

### 9.2 家庭规则
- 家庭的业务作用为：`结算时按家庭合并转账`
- 一个家庭中可包含多个成员。
- 一个家庭必须指定一名家主。
- 至少选择两名成员后，才允许组成家庭。

### 9.3 成员规则
- 第一版成员操作全部支持：
  - `保存昵称`
  - `禁用记账`
  - `清除认领`
  - `移出组队`

### 9.4 禁用记账规则
- 被禁用成员不能新增账单。
- 被禁用成员仍可查看账本。
- 被禁用成员仍可作为分摊对象。

### 9.5 清除认领规则
- 执行 `清除认领` 后，成员回退为 `待认领` 状态。
- 清除认领后保留临时成员身份。
- 清除认领不会将成员从成员列表中删除。

### 9.6 成员卡片字段要求
- 头像/首字头像
- 昵称展示位或输入位
- 身份标签
- 账单数
- 记账权限状态
- 微信绑定状态
- 操作按钮可用态/禁用态

## 10. 验收标准
- 页面结构与截图一致，包含 5 个固定区块。
- 邀请设置区包含邀请状态、开关与说明文案。
- 添加成员区支持空昵称校验。
- 家庭管理区明确要求至少两名成员且需要指定家主。
- 成员卡片完整展示身份、绑定、认领、记账权限等状态。
- `管理员` 与 `创建者` 视为同一角色的规则在文档中明确写出。
- `禁用记账`、`清除认领`、`移出组队` 的结果都有明确说明。
- 文档可直接供其他会话实施，无需再补关键业务边界。

- 当前界面视觉文案仍采用“猫猫”包装，实施时若保持现有 UI 风格，可继续沿用界面原词。
- 本文档用于交接实施，优先写清规则与边界，不展开未确认的扩展流程。
